# EXCLUSION PROMOTION CANDIDATE — INTERIM

Status:
Technical conclusion for KROK 65.

Purpose:
Record which current narrow exclusion behaviors
are already credible promotion candidates,
and which still must remain below contract-facing status.

---

## Promotion candidates

At the current Stage 5 level,
the following are credible promotion candidates:

1. complete duplicate identity is required before exclusion-related rejection
2. partial similarity must not trigger exclusion rejection
3. stabilization-stage rejection must remain explicit and auditable
4. canonical stabilized output must remain uniqueness-preserving
   at the rejection tick in the already validated narrow sense

Why these qualify:
- they are validator-backed
- they are already treated as narrow compliance expectations
- they are stable across the current control set
- they can be expressed without smuggling in broader mechanics

---

## Not yet promotable

The following still must remain below contract-facing promotion:
- annihilation interpretation
- fusion / bundling consequences
- re-projection conflict handling
- deeper-process claims before rejection
- any broad claim that exclusion is already fully implemented

Why these do NOT qualify:
- they are not closed by current Stage 5 work
- they would overstate present capability
- promoting them now would blur the line
  between validated narrow compliance and missing feature

---

## Technical conclusion

KROK 65 closes successfully.

The project now has an explicit promotion-candidate set for exclusion work.
That is stronger than the earlier boundary note:
- we no longer only know what must stay out,
- we now also know which narrow items are mature enough
  to be considered for future contract-facing promotion.

But this is still only candidacy.
It is not yet a formal contract promotion
and it is not yet a rewrite of `docs/ENGINE_CONTRACT.md`.

End.
