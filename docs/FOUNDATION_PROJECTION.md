## 1. Projection as Ontological Necessity

Observation is possible only in FRACTURE.
CORE has no:
- space,
- time,
- metric,
- objects.

Therefore, any observable structure must arise through a
**projection process** from CORE to FRACTURE.

Projection is:
- not an embedding,
- not a lossless mapping,
- not a coordinate transformation.

It is a **selective stabilization of relational structures**.

→ Ontology: THEORY_V3.9.md, Sections II–III

---

## 2. What Projection Is NOT

ROMION explicitly rejects the following interpretations:

- Projection is **not** coarse-graining of a hidden spacetime.
- Projection is **not** averaging over microscopic states.
- Projection is **not** probabilistic collapse.
- Projection is **not** dimensional reduction.

All of the above assume pre‑existing metric structure,
which CORE does not possess.

---

## 3. Stabilization and Visibility

Projection is governed by **stabilization**, not by observation.

A relational structure becomes observable if:
- it stabilizes sufficiently,
- its internal negotiation cost is low enough,
- it persists across relational updates.

Visibility is thus **a dynamical outcome**, not an observer effect.

---

## 4. Projection Thresholds

Practical projection in FRACTURE is mediated by three thresholds.
They do **not** define existence, only observability.

### 4.1 w_cluster — Object Threshold

Defines which relational groups stabilize into identifiable clusters.

- Above w_cluster:
  - relations form objects.
- Below:
  - relations remain unresolved.

Objects exist only in FRACTURE.

---

### 4.2 w_dist — Geometry Threshold

Defines which relations are used to construct background geometry.

- Only relations with:
  w_dist ≤ weight < w_bridge
  may participate in distance computation.

Background geometry:
- is derived,
- is local,
- has no global meaning.

---

### 4.3 w_bridge — Field / Channel Threshold

Defines relations acting as **bridges** between clusters.

Bridge relations:
- represent interaction channels,
- must never be used for metric geometry,
- are non‑spatial in nature.

---

## 5. Absolute Separation of Projection Layers

ROMION enforces strict separation:

- Object layer → w_cluster
- Geometry layer → w_dist
- Interaction layer → w_bridge

Mixing these thresholds:
- corrupts projection,
- creates spurious forces,
- produces invalid observables.

This is an **ontological safeguard**, not a numerical convention.

---

## 6. Projection Graphs

Projection induces two distinct derived graphs in FRACTURE:

### 6.1 Background Geometry Graph

- Nodes: clusters
- Edges: relations filtered by w_dist
- Purpose: distance, geometry, locality

### 6.2 Interaction / Bridge Graph

- Nodes: clusters
- Edges: relations filtered by w_bridge
- Purpose: fields, influence, propagation

Distances must **never** be computed on the bridge graph.

---

## 7. Incompleteness and Projection Loss

Projection is inherently lossy.

Consequences:
- CORE adjacency may project to distant positions.
- CORE locality may appear non‑local.
- Some relations never become observable.

This loss is not an error — it is fundamental.

Apparent paradoxes (non‑locality, dark sector, hidden structure)
arise from confusing projection artifacts with ontology.

---

## 8. Freeze and Thaw as Projection Phenomena

Freeze states emerge when:
- no bridge relations satisfy w_bridge,
- or active interaction weight drops to zero.

Frozen configurations:
- still exist relationally,
- no longer support observable dynamics.

Freeze is therefore:
- a projection failure,
- not annihilation,
- not a low‑quality result.

→ Ontology: THEORY_V3.9.md, Section II.4

## 9. Relationship to Measurement

Measurement does not alter CORE.
It only:
- samples projected structures,
- selects visible stabilization paths.

Measurement outcomes depend on:
- projection thresholds,
- stabilization history,
- phase of the CORE/FRACTURE boundary.

Observer‑centric interpretations are unnecessary.
