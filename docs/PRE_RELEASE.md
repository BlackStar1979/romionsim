# ROMION Documentation — Pre-Release Notice

Documentation Status: v1-prerelease  
Ontology Version: THEORY_V3.9

## Purpose of This Pre-Release

This documentation set is published as a pre-release
pending the availability of an approved reference simulation engine
(named romionsim).

A prototype or partial engine may exist in the repository
for technical validation of selected fragments of theory.
Such code does not end prerelease status by itself.

The ontology, architecture, and documentation structure
are considered stable.
What is missing is a verified executable implementation.

---

## What Is Considered Stable

The following elements are stable and locked:

- Ontology defined in THEORY_V3.9.md
- Epistemic separation and documentation architecture
- Definition of CORE, FRACTURE, and emergence rules
- Proper separation between:
  ontology, potentials, mappings, and hypotheses
- Documentation governance rules

No changes to these elements are expected
as part of the prerelease status.

---

## What Is Not Yet Complete

The following elements are intentionally incomplete:

- Approved reference simulation engine implementing CORE and FRACTURE
- Executable tests of hypotheses using romionsim
- End-to-end validation of documentation against code behavior

Until these exist at release quality, the documentation remains prerelease.

---

## Criterion for First Stable Release (v1)

The prerelease status will be lifted when:

- An approved reference Python engine named romionsim exists
- The engine implements the canonical ontology
- At least one HYPOTHESIS document can be tested end-to-end
- Documentation and code are consistent

At that point, the same documentation set
will be released as Documentation Release v1
without structural changes.

---

## Notes

This prerelease status reflects implementation readiness,
not theoretical uncertainty.

The ontology is considered complete
at the current version.

---

End of PRE-RELEASE
