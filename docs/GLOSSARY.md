# ROMION Glossary

## Role

Terminology reference for the ROMION documentation ecosystem.

This glossary:
- standardizes language across documents,
- prevents semantic drift,
- provides quick reference for readers and contributors.

It does **not** define ontology.
Whenever a term has ontological meaning, this glossary **points to the canonical THEORY document**.

---

## Governing Rules

1. Ontological authority resides exclusively in **THEORY_V3.9.md**.
2. This glossary does not introduce entities, rules, or mechanisms.
3. Definitions here are linguistic and contextual, not axiomatic.
4. In case of conflict, **THEORY overrides GLOSSARY**.

---

## Core Terms

### Romion
Elementary relational unit.
A node in the relational configuration Δ(t), not an object.
→ Ontology: THEORY_V3.9, Section I.1

---

### Δ (Delta) / Configuration
Total relational state at a given evolution step.
Represented as a (hyper)graph of relations.
→ Ontology: THEORY_V3.9, Sections I–II

---

### CORE
Non‑emergent relational phase.
Contains no space, time, objects, or metrics.
→ Ontology: THEORY_V3.9, Section II.1

---

### FRACTURE
Emergent phase in which stable relational patterns arise.
Only layer directly accessible to observation.
→ Ontology: THEORY_V3.9, Section II.2

---

### CORE / FRACTURE Boundary
Thin, active interface generating emergence.
Not a threshold but an engine of stabilization and selection.
→ Ontology: THEORY_V3.9, Section II.3

---

### delta_zero (Δ∅)
Limit attractor of CORE evolution corresponding to minimal contradiction.
Not directly observable.
→ Ontology: THEORY_V3.9, Section II.1.a

---

## Structural Terms

### Δ‑Loop
Closed relational path in the projected graph.
Represents a stable relational resonance.
Not an object.
→ Ontology: THEORY_V3.9, Section III.1

---

### Loop Stability μ(C)
Structural stability measure of a Δ‑loop based on coherence and weights.
Sometimes informally referred to as “mass proxy”.
→ Ontology: THEORY_V3.9, Section III.2

---

### Loop Orientation σ(C)
Binary orientation of a Δ‑loop (+1 or −1).
Used in defining topological charge and exclusion.
→ Ontology: THEORY_V3.9, Section III.2

---

### Topological Charge Q_T
Parity‑based charge derived from loop length and orientation.
→ Ontology: THEORY_V3.9, Section III.2

---

### Niche (Topological Niche)
Pair defining loop placement:
(core_cluster, excitation index n).
Used for state exclusion.
→ Ontology: THEORY_V3.9, Section III.3

---

## Matter and States

### Material Entity
Equivalence class of Δ‑loop states sharing invariants and niche.
No primitive particles exist in ROMION.
→ Ontology: THEORY_V3.9, Section III.3

---

### Particle (informal)
Domain‑level name for a material entity used in external mappings.
Has no ontological authority inside THEORY.
→ Domain mapping only (MAP_* documents)

---

## Constraints and Rules

### Topological Exclusion Principle (Pauli Rule)
Constraint preventing identical loop states from occupying the same niche.
Structural, not probabilistic.
→ Ontology: THEORY_V3.9, Section III.4

---

### Freeze State
FRACTURE‑level condition with no active field structure.
Defines phase boundaries, not outcomes.
→ Ontology: THEORY_V3.9, Section II.4

---

### Boundary Claim
Statement about phase transition or boundary location.
Valid only under strict criteria.
→ Ontology: THEORY_V3.9, Section VI.13

---

## Methodological Terms

### Projection
Process by which CORE structure becomes observable in FRACTURE.
Not an embedding or mapping in space.
→ FOUNDATION_PROJECTION.md

---

### Thresholds (w_cluster, w_dist, w_bridge)
Projection constraints defining objects, geometry, and fields.
Must not be mixed.
→ FOUNDATION_PROJECTION.md

---

### Diagnostic Metric
Exploratory metric without ontological authority.
Diagnostics never define entities or boundaries.
→ THEORY_V3.9, Section VI.9

---

### Fail‑Closed
Principle stating that invalid data invalidates an entire result.
→ THEORY_V3.9, Section VI.10

## Usage Note

When introducing a new term:
- If it defines *existence*, it belongs in THEORY.
- If it extends structure, it belongs in SPEC.
- If it names external physics, it belongs in MAP.
- If it is testable and falsifiable, it belongs in HYPOTHESIS.
